ROS Assessment Tool Prototype for Deliverable (D3.1)

Name

ROS Assessment Tool - Assesses the quality of the Research objects. At the current stage the tool uses xml files of the ROS. These xml files are generated by pdfx. To test the tool, please download the xml 40/80 paper xml dataset from redmine and extract it to your preferred folder.

Prerequisites

  1. You should have Mysql up and running on your local machine. You need to have access to create a new database and should be able to run it on the local host.
  2. You should have Scopus API Key. If your institution is a subscriber of this service, you can directly request for the API key
  3. Java jdk 1.6 or latest version installed and running on your machine

Options

Option 1: running the application using the jar files

Steps:
  1. download the jar file from ROSAT JAR . This file contains the jar file and all the associated dependent files and libraries.
  2. Extract the ROSAT.Zip to your preferred location.
  3. Open the file config.xml file with your preferred editor. Make sure that the config.xml file is located in the same folder where the jar files and lib files are stored
  4. Enter your preferred settings for each of the entries
    1. ros_repository_location is the folder where the xml dataset is stored
    2. citation_data_location. The location of the initial data where you store the downloaded sjr and jcr. If you don't have these data set, please use the default location.
    3. ros_qa_output_location is the folder where the resulting csv and arff files will be stored
    4. scopus_api. Put your Scopus api here
    5. db_name. Put your preferred database name here. Default name is DrInventor
    6. db_user_name. Put your database username here. Default user is root.
    7. db_pass_word. Put your preferred database password here. Default name is ""(empty string)
  5. On your command prompt go to the folder you extract the zip file and type:
    java -cp rosat.jar ie.nuim.cs.dri.repository.Repository
    The Program will create the database, create all the required tables and populate them with the required data. This may take for a while.
  6. On your command prompt type:
    java -cp rosat.jar ie.nuim.cs.dri.metadata.MetadataExtractor
    The program will read the research objects form the repository location and collect citation data from Scopus and other sources and store the results as csv and arff format under the ros_qa_output_location

Option 2: running the application from the source file using NetBeans

Steps:
  1. Download the source file from ROSAT Source.
  2. Extract the ROSAT.Zip to your preferred location. Open the file config.xml file with your preferred editor. Make sure that the config.xml file is located in the same folder where the src files are stored
  3. Enter your preferred settings for each of the entries
    1. ros_repository_location is the folder where the xml version of the RO is stored
    2. citation_data_location. The location of the initial data where you store the downloaded sjr and jcr. If you don't have this data set, please use the default location.
    3. ros_qa_output_location is the folder where the resulting csv and arff files will be stored
    4. scopus_api. Put your Scopus api here
    5. db_name. Put your preferred database name here. Default name is DrInventor
    6. db_user_name. Put your database username here. Default user is root.
    7. db_pass_word. Put your preferred database password here. Default name is ""(empty string)
  4. Under the ie.nuim.cs.dri.repository package, run the Repository.java
  5. Under the ie.nuim.cs.dri.metadata package, run the MetadataExtractor.java file

How to report bugs

This is an initial implementation of the ROS assessment tool and by no means is complete. We expect more comments and bug reports from other groups. Please report any bug or forward your questions to abgaz.yalemisew@nuim.ie or report it on redmine.
